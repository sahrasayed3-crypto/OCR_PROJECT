# Commercial Components Policy

## Private Components

The following components are not part of the public open-source grant unless explicitly released under a separate written license:

- training datasets and private reference texts
- final OCR model weights
- LoRA / QLoRA adapters
- checkpoints and intermediate training artifacts
- exact training recipes, final hyperparameters, and private evaluation thresholds
- advanced proprietary synthetic distortion pipelines
- private data-collection and permission-management tools
- production service code and hosted model infrastructure
- API keys, credentials, cloud configuration, and deployment secrets
- customer data, uploaded PDFs, generated DOCX output, logs, and databases
- trademarks, logos, brand assets, and commercial names

## Separate Commercial Licensing

The project owner may offer separate commercial licenses in the future for hosted services, private model access, model weights, adapters, data, support, or enterprise deployment. These commercial offerings are not included in the public repository license.

## Hosted Or Paid Services

Paid APIs, hosted OCR models, managed conversion services, cloud deployments, or commercial support are not automatically granted by the open-source source-code license.

## Data And Weights

Any dataset, model weight, adapter, or checkpoint release must include its own license, usage scope, redistribution terms, and privacy review. The public code license alone is not permission to use private data or weights.

## Brand And Trademark

The source-code license does not automatically grant permission to use project names, logos, visual identity, or trademarks in a way that implies endorsement or official affiliation.

## Commercial Licensing Contact

For future commercial licensing inquiries, publish a dedicated contact route such as:

```text
Commercial licensing: licensing@example.com
```

Do not publish personal email addresses, private phone numbers, customer details, or sensitive business records in this repository.
