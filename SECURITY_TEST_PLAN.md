# خطة الاختبار الأمني

## البوابات

1. تحليل AST وبحث sinks والأنماط الخطرة.
2. Bandit وRuff وmypy وcompileall، مع تسجيل الأدوات غير المتاحة.
3. تدقيق التبعيات وSBOM وسلسلة التوريد وGitHub Actions.
4. فحص الأسرار والملفات الممنوعة والحزم المبنية.
5. اختبارات حدود المسارات والروابط وWindows path semantics.
6. مثبتات PDF وصور وXML وZIP مشوهة ومحدودة الحجم.
7. اختبارات FastAPI محلية للمصادقة والحجم والتكرار والرؤوس.
8. اختبارات SQLite والتزامن وRedis/RQ إن كان Redis محليًا متاحًا.
9. اختبارات command injection وSSRF ضد خادم localhost محكوم.
10. اختبارات النموذج وOCR وDOCX وCSV وHTML.
11. اختبارات الترخيص وسمّ البيانات ومنع وثائق المستخدم من التدريب.
12. اختبارات النسخ والاستعادة والتنظيف وانخفاض القرص.
13. 30 سيناريو قبول عدائي اصطناعي.
14. pytest كامل وتغطية وبناء wheel/sdist وتثبيت معزول وCLI/API smoke.

## سياسة الإثبات

لكل ثغرة مؤكدة: اختبار فاشل أو استنساخ محلي، أصغر إصلاح صحيح، اختبار
ارتداد، تحقق مستهدف، تحقق كامل، وتوثيق الخطر المتبقي. لا تُعد النتيجة
ثغرة مؤكدة إن كانت مجرد مطابقة نصية بلا مسار استغلال.

## حدود السلامة

- الملفات التجريبية صغيرة، وعدد الأعضاء والصفحات والإطارات محدود.
- لا ضغط أو fuzzing غير محدود.
- لا يُستخدم عنوان غير loopback في الاختبارات الديناميكية.
- لا تُطبع قيم حساسة؛ تستخدم العلامة الاصطناعية
  `CLOUDA-SECURITY-SENSITIVE-MARKER`.
