# Data Licenses

The repository is prepared for public open-source review with a conservative data posture. It should contain only deterministic fixtures, small generated samples, and documentation whose license status is clear.

## Allowed In Repository

- Small synthetic or generated PDF fixtures used by tests.
- Reference text files for deterministic sample cases.
- Documentation describing planned data collection and evaluation methodology.
- License metadata, manifests, and hashes for future datasets.

## Not Allowed In Repository

- Copyrighted books or scans without written permission.
- Private email permission evidence containing personal data.
- Training datasets whose redistribution rights are unclear.
- Model weights, checkpoints, embeddings, or large generated outputs.
- Uploaded user PDFs, generated DOCX files, logs, databases, backups, or caches.

## Current Dataset Status

- Public test fixtures are intended to be deterministic and copyright-free.
- Scanned-page OCR accuracy is not claimed.
- CER and WER have not been measured for a selected final OCR model.
- Any future benchmark dataset must record source, license, permission status, split, hash, and allowed usage.

## Permission Evidence

Permission emails or private correspondence should be stored outside the public repository if they include personal data or private details. Public documentation may summarize the permission status without exposing private contact information.
