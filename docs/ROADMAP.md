# Roadmap

## Next 0–2 months

- Maintain the direct-text conversion path and metadata contract.
- Curate a consented, versioned ground-truth evaluation set.
- Define acceptance criteria for Arabic, English, mixed-direction, old, and degraded documents.

## Months 2–4

- Evaluate candidate OCR engines behind the model-agnostic interface.
- Measure accuracy, latency, memory, and failure behavior on representative hardware.
- Decide whether an optional CPU and/or AMD-compatible deployment path is viable.

## Months 4–6

- Integrate only the selected, validated engine as an optional component.
- Publish reproducible evaluation results and hardware/software details.
- Improve editable DOCX structure while retaining page provenance and manual-review routes.

No roadmap item is a claim that a final OCR model, GPU execution, or accuracy benchmark already exists.
